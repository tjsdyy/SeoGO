# 关键词布局

> 关键词分析完成后，下一步是将关键词科学地"布局"到网站的各个层面——从域名选择到 URL 结构，从 Title 标签到正文密度。关键词布局的核心原则是：**让搜索引擎和用户都能清晰地理解每个页面的主题，同时避免过度优化被惩罚**。

---

## 一、域名选择策略

### 1.1 域名对 SEO 的影响

域名本身对 SEO 的直接影响已经很小，但间接影响不可忽视：

| 影响维度 | 说明 |
|---------|------|
| 品牌辨识度 | 好记的域名带来更多直接访问和品牌搜索 |
| 点击率 | 搜索结果中，简短清晰的域名更容易获得点击 |
| 外链获取 | 有品牌感的域名更容易获得其他网站的引用 |
| 用户信任 | .com 域名的用户信任度普遍高于其他 TLD |

### 1.2 三种域名命名策略

**策略一：品牌词域名（最推荐）**

创造一个独特的品牌词，如 Ahrefs、Canva、Notion。

- 优点：独特性强、商标保护容易、不受关键词限制
- 适用场景：长期品牌建设、SaaS 产品、想要打造品牌溢价

**策略二：组合词域名**

将两个有意义的短词组合，如 MailChimp、WordPress、YouTube。

- 优点：有含义、好记、通常有 .com 可用
- 适用场景：想要域名暗示业务方向，但又不纯粹堆砌关键词

**策略三：关键词 + 修饰词域名**

在行业关键词基础上加修饰词，如 SmartSEO、RankTracker。

- 优点：直观表达业务，新访客容易理解
- 缺点：品牌感较弱，域名可能已被注册

### 1.3 域名命名的禁区

| 禁区 | 原因 |
|------|------|
| 纯关键词域名（如 best-seo-tools.com） | Google 已降低 EMD（Exact Match Domain）的权重 |
| 包含连字符的域名（如 my-brand-name.com） | 看起来不专业，用户难记 |
| 过长的域名 | 难以记忆和输入，控制在 15 个字符以内 |
| 与知名品牌相似的域名 | 可能涉及商标侵权，且不利于建立独立品牌 |

### 1.4 TLD（顶级域名）选择

| TLD | 适用场景 | 推荐度 |
|-----|---------|--------|
| **.com** | 通用商业网站 | 强烈推荐 |
| **.io** | SaaS、开发者工具 | 推荐 |
| **.ai** | AI 工具和服务 | 推荐（AI 领域） |
| **.dev** | 开发者工具（强制 HTTPS） | 推荐 |
| **.net / .org** | 技术类 / 社区类 | 一般 |
| **ccTLD**（.us/.uk/.de） | 针对特定国家市场 | 特定场景推荐 |

> **建议**：如果能拿到 .com，优先选择 .com。避免使用冷门 TLD（如 .xyz、.club），用户信任度较低。

## 二、关键词密度规划

### 2.1 什么是关键词密度

**关键词密度 = 关键词出现次数 / 页面总字数 x 100%**

关键词密度是传统 SEO 的概念，虽然 Google 官方表示不使用关键词密度作为排名因素，但合理的关键词分布仍然有助于搜索引擎理解页面主题。

### 2.2 推荐密度参考

| 关键词类型 | 推荐密度 | 2000 词文章中的次数 |
|-----------|---------|-------------------|
| 主关键词 | 1-2% | 自然出现 10-20 次 |
| 次要关键词 | 0.5-1% | 各出现 5-10 次 |
| 长尾关键词 | 0.3-0.5% | 各出现 3-5 次 |

**核心原则**：密度只是参考指标，**自然度永远优先于密度数字**。如果一个关键词出现 15 次但读起来都很自然，那就没问题；如果出现 8 次但感觉生硬，那就是过度优化。

### 2.3 TF-IDF 分析法

TF-IDF（Term Frequency - Inverse Document Frequency）是比"关键词密度"更科学的内容优化方法。它通过分析 SERP 排名靠前的页面，找出这些页面共同使用的"语义相关词"。

**工作原理**：
- **TF（词频）**：一个词在你的页面中出现的频率
- **IDF（逆文档频率）**：这个词在整个 SERP 中的稀缺程度
- **TF-IDF 得分高的词**：出现在排名靠前的页面中、但不是每个页面都有的"差异化"用词

**实操工具**：

| 工具 | 特点 | 价格 |
|------|------|------|
| Surfer SEO | Content Editor 直接给出推荐词和密度 | $89/月起 |
| Clearscope | 语义分析最精确 | $170/月起 |
| Frase | 性价比较高，AI 辅助写作 | $15/月起 |
| MarketMuse | 企业级内容优化 | $149/月起 |

**操作步骤**：

1. 在工具中输入目标关键词
2. 工具自动分析 SERP 前 10-20 名页面的用词
3. 生成一份"推荐词汇列表"及其建议出现次数
4. 在写作过程中自然覆盖这些推荐词
5. 完成后用工具的评分功能检查内容得分

### 2.4 避免关键词堆砌

Google 明确将关键词堆砌列为违规行为。以下是常见的堆砌表现：

```
❌ 关键词堆砌示例：
"我们的 SEO 工具是最好的 SEO 工具。如果你在找 SEO 工具，
我们的 SEO 工具可以帮你做 SEO。使用我们的 SEO 工具来提升 SEO 效果。"

✅ 自然写作示例：
"我们的搜索引擎优化平台集成了关键词分析、排名追踪和站点审计功能。
无论你是新手还是资深优化师，都能用它快速发现网站的 SEO 问题并制定改进方案。"
```

**判断标准**：把文章读出声来，如果某个词让你觉得"又出现了"，那就是密度过高了。

## 三、URL 结构优化

### 3.1 SEO 友好的 URL 原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 简短 | 越短越好，控制在 60 个字符以内 | `/seo-tools/` |
| 包含关键词 | URL 中包含目标关键词 | `/best-crm-software/` |
| 使用连字符 | 单词之间用连字符分隔 | `/keyword-research-guide/` |
| 全小写 | 避免大小写混用 | `/blog/seo-tips/` |
| 避免参数 | 不使用 ?id=123 形式 | 用 `/product/name/` 代替 |
| 扁平结构 | 目录层级不超过 3 层 | `/category/subcategory/page/` |

### 3.2 URL 关键词策略

```
首页：
  example.com
  → 不需要在 URL 中加关键词，靠 Title 和内容优化

分类页：
  example.com/seo-tools/
  → 包含分类核心关键词

产品/内容页：
  example.com/seo-tools/keyword-research-tool/
  → 包含页面级关键词

博客文章：
  example.com/blog/how-to-do-keyword-research/
  → 包含文章目标关键词
```

### 3.3 URL 结构常见错误

```
❌ 错误示例：
example.com/p?id=12345
example.com/2025/03/17/this-is-a-very-long-url-with-too-many-words-in-it
example.com/SEO_Tools/Best-Tools
example.com/category1/subcategory2/sub-sub3/sub-sub-sub4/page

✅ 正确示例：
example.com/seo-tools/
example.com/blog/keyword-research-guide/
example.com/pricing/
```

## 四、Title Tag 关键词布局

### 4.1 Title Tag 的重要性

Title Tag 是站内 SEO 中最重要的单一元素。它直接出现在搜索结果中，同时影响排名和点击率。

### 4.2 Title Tag 关键词规则

| 要素 | 建议 |
|------|------|
| 长度 | 50-60 个字符（约 600 像素宽度） |
| 关键词位置 | 主关键词尽量出现在前半部分 |
| 品牌名 | 放在末尾，用竖线或横杠分隔 |
| 唯一性 | 每个页面的 Title 必须唯一 |
| 吸引力 | 需要在 SERP 中吸引用户点击 |

### 4.3 不同页面类型的 Title 公式

| 页面类型 | Title 公式 | 示例 |
|----------|-----------|------|
| 教程文章 | How to + 主关键词 + (补充) | How to Do Keyword Research (7 Steps) |
| 列表文章 | 数字 + Best + 关键词 + 年份 | 11 Best SEO Tools for 2025 |
| 深度指南 | 关键词: The + 修饰 + Guide | Link Building: The Definitive Guide |
| 对比页面 | A vs B: 对比维度 | HubSpot vs Salesforce: Features & Pricing |
| 产品页面 | 产品名 - 核心卖点 \| 品牌 | RankTracker - Real-Time SEO Rankings \| Brand |
| 落地页 | 主关键词 + 价值主张 | Free SEO Audit Tool - Check Your Site in 60s |

### 4.4 Title 优化技巧

1. **使用数字**：含数字的 Title 比不含数字的 CTR 高 36%
2. **包含年份**：如 "2025" 传递内容新鲜度信号
3. **使用括号**：如 "(Free Template)"、"(Step-by-Step)" 提升 CTR
4. **情感触发词**：Ultimate、Proven、Essential、Complete
5. **测试优化**：在 Search Console 中找到 CTR 低的页面，修改 Title 后观察 2-4 周

## 五、Meta Description 关键词布局

### 5.1 基本规则

| 要素 | 建议 |
|------|------|
| 长度 | 120-155 个字符 |
| 关键词 | 自然包含主关键词（匹配词会被加粗显示） |
| CTA | 结尾包含行动号召 |
| 唯一性 | 每个页面的 Description 必须唯一 |
| 准确性 | 准确描述页面内容，不误导用户 |

> **注意**：Meta Description 不是直接排名因素，但它影响 CTR。搜索用户在 SERP 中看到你的 Description，如果其中包含他们搜索的关键词（加粗显示），会更倾向于点击。

### 5.2 不同意图的 Description 模板

**信息型内容**：
```
Learn [what/how] [主题] with our [year] guide. Covers [要点1], [要点2],
and [要点3]. [CTA].
```

**商业调查型内容**：
```
Looking for [产品/服务]? Compare [数量] top [类型] based on [维度1],
[维度2] & [维度3]. [CTA].
```

**交易型内容**：
```
[产品名] starting at $[价格]. [核心卖点]. [信任信号]. [CTA].
```

**工具型内容**：
```
Use our free [工具名] to [动作]. [好处]. No signup required. [CTA].
```

## 六、关键词布局全局规划

### 6.1 网站层级与关键词映射

```
首页
  → 核心品牌词 + 1-2 个行业大词
  → 示例：Brand - Best SEO Tools & Software

分类页
  → 中部词（Body Terms）
  → 示例：/seo-tools/ → "SEO Tools"

子分类页
  → 更具体的中部词
  → 示例：/seo-tools/keyword-research/ → "Keyword Research Tools"

产品/内容页
  → 长尾关键词
  → 示例：/blog/best-free-keyword-research-tools/ → "best free keyword research tools"

FAQ / 帮助页
  → 问答型长尾词
  → 示例：/faq/how-to-check-keyword-difficulty/ → "how to check keyword difficulty"
```

### 6.2 关键词去重原则

**一个核心原则：一个关键词只对应一个页面。**

如果多个页面都在竞争同一个关键词，就会产生"关键词自相残杀"（Keyword Cannibalization），导致：
- Google 不知道该排哪个页面
- 排名在多个页面之间波动
- 每个页面都无法获得最佳排名

**处理方法**：
1. 在关键词表格中为每个关键词指定唯一的目标页面
2. 如果发现多个页面在竞争同一个词，合并内容或用 301 重定向
3. 使用 Canonical 标签明确指定首选页面

## 七、本节检查清单

- [ ] 域名简短、好记、有品牌感，优先选择 .com
- [ ] 关键词密度控制在 1-2%，使用 TF-IDF 工具辅助优化
- [ ] URL 结构简短、含关键词、使用连字符、全小写
- [ ] 每个页面的 Title Tag 唯一且包含目标关键词
- [ ] 每个页面的 Meta Description 唯一且包含关键词和 CTA
- [ ] 完成网站层级与关键词的全局映射
- [ ] 确认无关键词自相残杀问题
