# 站点方向选择

> 确定了站点类型后，下一步是选择站点方向：套壳、聚合站还是品牌站。三种方向在启动速度、投入成本和长期价值上有本质区别。本节帮你做出最适合当前阶段的选择。

---

## 三种方向对比

| 维度 | 套壳 | 聚合站 | 品牌站 |
|------|------|--------|--------|
| 启动速度 | 快（1-4 周） | 中（4-8 周） | 慢（2-6 月） |
| 投入成本 | 低 | 低-中 | 中-高 |
| 品牌溢价 | 极低 | 低 | 高 |
| 护城河 | 几乎没有 | 内容积累 | 品牌心智 + 用户忠诚 |
| SEO 难度 | 中 | 中 | 初期高，后期回报大 |
| 长期价值 | 低 | 中 | 高（可被收购） |

---

## 套壳产品

套壳产品（Wrapper Product）是在现有 API 或开源项目基础上，重新包装界面和营销推向市场的产品形态。这是很多出海团队的起步方式。

### 常见形态

| 形态 | 说明 | 典型案例 |
|------|------|----------|
| API 套壳 | 调用 OpenAI 等 API，封装成独立产品 | 各类 AI 写作/绘画工具 |
| 开源项目套壳 | 基于开源项目二次开发 | 基于 Whisper 的语音转写工具 |
| 白标/贴牌 | 购买白标授权，换 Logo 运营 | 部分 CRM、邮件营销工具 |
| 功能精简版 | 将复杂产品某功能拆出做独立工具 | 从 Photoshop 拆出的"去背景"工具 |

### 优势

- **启动速度快**：1-4 周上线 MVP，快速验证市场需求
- **技术门槛低**：前端 + API 对接即可，一个全栈开发者可独立完成
- **SEO 友好**：围绕具体长尾关键词构建产品页面，快速卡位

### 风险

- **同质化严重**：API 火爆后短时间涌现大量套壳产品
- **依赖上游**：API 可能涨价、限流、关闭服务
- **法律合规**：部分 API 明确禁止直接转售套壳
- **护城河薄弱**：竞争对手可以快速复制

### SEO 策略

```
❌ 不推荐：AI writer（竞争极其激烈）
✅ 推荐：AI product description generator for Shopify
✅ 推荐：free AI email subject line generator
```

核心思路：**工具名 + 使用场景 + 平台/行业 = 精准长尾词**

### 页面结构

```
首页
├── /tools/[具体工具名]     ← 各功能独立页面
├── /use-cases/[使用场景]   ← 场景页
├── /compare/[竞品名]       ← 对比页
├── /blog/[教程文章]        ← 内容营销
└── /pricing                ← 定价页
```

### 进化路径

```
阶段 1：纯套壳 → 快速上线，验证需求
阶段 2：增值套壳 → 加入独有功能、数据积累
阶段 3：半自研 → 核心模块自研，API 为辅
阶段 4：全自研 → 建立技术壁垒，形成护城河
```

> 套壳是手段，不是目的。用套壳快速验证市场，然后逐步建立核心竞争力。

### 成功案例

- **Remove.bg**：图像背景移除 API 套壳 → 极致用户体验 → 被 Canva 收购
- **Monica.im**：ChatGPT API 浏览器插件 → 浏览器原生集成 → 数百万用户

## 聚合站

聚合站通过整合信息、组织内容获取搜索流量，再通过广告、联盟营销或导流变现。是个人站长和小团队的高性价比选择。

### 核心优势

1. **天然 SEO 优势**：海量页面覆盖长尾词，结构化数据适合 Schema Markup
2. **启动成本低**：不需自研产品，一个人即可运营
3. **变现路径多样**：广告、联盟营销、付费收录、赞助内容、导流佣金

### 内容矩阵设计

覆盖用户搜索旅程的完整漏斗：

```
认知阶段 → "what is [概念]" 类文章
    ↓
调研阶段 → "best [品类]" 列表 + "[产品] review" 评测
    ↓
对比阶段 → "[A] vs [B]" 对比 + "[产品] alternatives"
    ↓
决策阶段 → "[产品] pricing" + "[产品] discount/coupon"
    ↓
使用阶段 → "how to use [产品]" 教程
```

### 技术要点

- **抓取效率**：合理 URL 结构，避免参数化 URL 重复抓取
- **Canonical 标签**：处理相似页面
- **结构化数据**：为产品添加 Schema Markup（评分、价格等）
- **内链策略**：分类页→子页面、详情页互相关联、对比页双向链接

### 常见陷阱

1. 直接复制产品官网描述 → 重复内容惩罚，必须原创重写
2. 空壳页面只有基本信息 → 确保每个页面有独特价值
3. 联盟链接过多 → 保持合理的内容与链接比例

### 运营路线图

```
第 1-4 周：选定领域，搭建框架
第 5-8 周：录入首批 50-100 个产品
第 9-12 周：发布 20-30 篇高质量内容
第 13-16 周：外链建设 + Google 收录
第 4-6 月：持续更新，优化排名
第 6-12 月：开始变现，优化转化率
```

## 品牌站

自有品牌出海投入更大、周期更长，但一旦建立品牌认知，将拥有最深的护城河和最高的溢价空间。

### 品牌定位公式

```
对于 [目标用户群]，
[品牌名] 是 [品类] 中的 [差异化定位]，
因为 [支撑理由]。
```

**实例**：

```
对于需要快速生成营销文案的中小电商卖家，
CopySnap 是 AI 文案工具中的"电商专用文案引擎"，
因为它内置了 50+ 电商场景模板和 A/B 测试功能。
```

### 差异化策略

| 差异化路径 | 说明 | 适用情况 |
|------------|------|----------|
| 场景聚焦 | 专注某个特定使用场景 | 大品牌做得太泛时 |
| 人群聚焦 | 专门服务某个细分人群 | 被主流产品忽视的群体 |
| 价格差异 | 更低价或更灵活定价 | 现有产品定价过高时 |
| 体验差异 | 更简单、更快、更美 | 现有产品太复杂时 |
| 地区差异 | 针对特定地区本地化 | 全球产品忽视某些市场时 |

### 品牌命名原则

- 简短易记（2-3 音节最佳）
- 英语发音无障碍，避免歧义
- .com 域名可获取（或 .io、.ai）
- 在目标市场无商标冲突

### 品牌 SEO 策略

**品牌词建设**：确保用户搜索品牌名时，搜索结果第一页全部可控。

**内容层级**：

```
第 1 层：品牌核心页面
├── 首页（品牌定位 + 核心卖点）
├── About 页（品牌故事 + 团队）
├── 产品/功能页
└── 定价页

第 2 层：搜索流量页面
├── 博客文章（信息类关键词）
├── 使用场景页
├── 客户案例（Case Study）
└── 对比页（vs 竞品）

第 3 层：长尾覆盖
├── 帮助中心/知识库
├── 模板/资源库
└── 行业/地区子页面
```

### 外链建设

| 方式 | 操作 | 效果 |
|------|------|------|
| PR 报道 | 向科技媒体投稿发布新闻 | 高权重 dofollow 链接 |
| 行业评测 | 邀请 KOL 撰写评测 | 品牌曝光 + 外链 |
| 嘉宾发文 | 在行业博客发署名文章 | 稳定 dofollow 外链 |
| 产品目录 | 提交到 Product Hunt 等 | 初期外链基础 |
| 开源贡献 | 开源部分工具/代码 | GitHub 外链 + 信任 |

### 品牌建设时间线

| 阶段 | 时间 | 核心任务 | SEO 重点 |
|------|------|----------|----------|
| 品牌奠基 | 0-3 月 | 命名、VI、官网上线 | 品牌词占位 |
| 内容启动 | 3-6 月 | 发布 30-50 篇内容 | 长尾词排名 |
| 流量增长 | 6-12 月 | 外链建设、社媒运营 | 核心词排名提升 |
| 品牌扩张 | 12-24 月 | PR 报道、行业合作 | 高竞争度词排名 |
| 品牌成熟 | 24+ 月 | 品牌词流量占 30%+ | 品牌自带流量 |

### 成功案例

- **Notion**：模板库覆盖海量长尾词 + 用户社区内容 + 极简美学
- **Canva**：免费工具矩阵 + 100+ 语言本地化 + "让非设计师做专业设计"
- **CapCut**：围绕 "free video editor" 做内容 + TikTok 生态绑定

## 方向选择决策框架

根据你的当前阶段选择：

```
如果你是：
├── 零经验、想快速试水 → 套壳（1-4 周上线验证）
├── 个人站长、有内容能力 → 聚合站/导航站（4-8 周搭建）
├── 有行业经验、想长期做 → 品牌站（2-6 月投入）
└── 不确定 → 先套壳验证需求，再升级为品牌站
```

> **核心建议**：不必一步到位。可以从套壳起步快速验证，验证成功后逐步升级为品牌站。关键是选一个方向立刻开始行动。
